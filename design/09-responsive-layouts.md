# Layouts Responsives

## Description

Spécifications des adaptations responsive pour les différents dispositifs et résolutions d'écran.

## Prompt IA pour Layouts Responsives

```
Create a comprehensive responsive layout guide for "Waajal Ëlëk" showing how the military pension platform adapts across different devices and screen sizes.

TITLE: "WAAJAL ËLËK - Guide des Layouts Responsives"
Subtitle: "Adaptation Multi-Dispositifs - Mobile First Design"

BREAKPOINTS TO SHOW:
- Mobile Portrait: 320px - 480px (primary focus)
- Mobile Landscape: 481px - 768px  
- Tablet Portrait: 769px - 1024px
- Tablet Landscape: 1025px - 1200px
- Desktop: 1201px - 1920px
- Large Desktop: 1921px+

DEVICE MOCKUPS TO INCLUDE:
1. iPhone 14 (393x852px) - Primary mobile reference
2. Samsung Galaxy S23 (360x800px) - Android reference  
3. iPad (768x1024px) - Tablet reference
4. iPad Pro (1024x1366px) - Large tablet
5. MacBook Pro (1440x900px) - Desktop reference
6. 4K Display (3840x2160px) - Large desktop

RESPONSIVE LAYOUTS TO DEMONSTRATE:

1. DASHBOARD LAYOUT ADAPTATION:
   Desktop (1920px):
   - Full sidebar navigation (280px)
   - Main content in 4-column grid
   - All widgets visible simultaneously
   - Hover interactions enabled

   Tablet (1024px):
   - Collapsible sidebar (60px collapsed, 280px expanded)
   - Main content in 2-column grid  
   - Priority widgets shown first
   - Touch-optimized interactions

   Mobile (393px):
   - Hidden sidebar, hamburger menu
   - Single column, stacked layout
   - Bottom tab navigation
   - Swipe gestures for navigation
   - Large touch targets (44px minimum)

2. ACCOUNT VIEW RESPONSIVE BEHAVIOR:
   Desktop:
   - Side-by-side layout (60% content, 40% sidebar)
   - Detailed charts with hover tooltips
   - Expanded table views with all columns

   Tablet:
   - Stacked layout with priority content first  
   - Simplified charts with touch interactions
   - Horizontally scrollable tables

   Mobile:
   - Fully stacked single-column
   - Card-based information display
   - Simplified chart views
   - Collapsible table sections

3. NAVIGATION PATTERN EVOLUTION:
   Desktop: Top horizontal navigation + left sidebar
   Tablet: Top navigation + collapsible sidebar
   Mobile: Bottom tab bar + hamburger menu

4. FORM LAYOUT ADAPTATIONS:
   Desktop: Multi-column forms (2-3 columns)
   Tablet: Two-column with smart field grouping
   Mobile: Single column, optimized field sizing

5. DATA TABLE RESPONSIVE STRATEGIES:
   Desktop: Full table with all columns visible
   Tablet: Priority columns + horizontal scroll
   Mobile: Card-based view or accordion table

TYPOGRAPHY SCALING:
- Desktop: Base font size 16px
- Tablet: Base font size 15px  
- Mobile: Base font size 14px (minimum)
- Heading scale adjustments per device
- Line height optimizations for readability

SPACING AND LAYOUT:
- Desktop: 32px, 24px, 16px spacing units
- Tablet: 24px, 16px, 12px spacing units
- Mobile: 16px, 12px, 8px spacing units
- Touch target sizing: 44px minimum on mobile
- Safe area considerations for notched devices

COMPONENT BEHAVIOR CHANGES:
1. Modals:
   - Desktop: Centered overlay
   - Tablet: Larger overlay, more padding
   - Mobile: Full-screen or bottom sheet

2. Dropdowns:
   - Desktop: Standard dropdown
   - Tablet: Touch-optimized dropdown
   - Mobile: Full-screen picker or bottom sheet

3. Charts and Graphs:
   - Desktop: Interactive with hover states
   - Tablet: Touch interactions, simplified
   - Mobile: Simplified view, swipe navigation

4. Sidebars:
   - Desktop: Persistent sidebar
   - Tablet: Overlay sidebar
   - Mobile: Bottom sheet or full-screen drawer

PERFORMANCE CONSIDERATIONS:
- Mobile: Optimized images, lazy loading
- Tablet: Balanced performance and features  
- Desktop: Full feature set, larger assets
- Progressive enhancement approach
- Touch vs mouse interaction patterns

SPECIFIC MOBILE OPTIMIZATIONS:
- Thumb-friendly navigation zones
- Swipe gestures for common actions
- Pull-to-refresh functionality
- Bottom-sheet modals for forms
- Large touch targets for financial actions
- One-handed usage considerations

FRENCH LANGUAGE ADAPTATIONS:
- Text overflow handling for longer French words
- Proper line breaking for compound words
- Cultural formatting for numbers and dates
- Right-to-left support considerations (if needed)

ACCESSIBILITY RESPONSIVE FEATURES:
- Scalable text (up to 200% zoom)
- High contrast mode adaptations
- Focus indicators sizing by device
- Screen reader optimizations per layout
- Keyboard navigation patterns

VISUAL DEMONSTRATION:
- Side-by-side device comparisons
- Before/after responsive adaptations
- Interactive element sizing guides
- Grid system overlays showing column usage
- Touch target sizing visualization

DESIGN SPECIFICATIONS:
- Show actual pixel dimensions
- Include spacing measurements
- Demonstrate content prioritization
- Loading state adaptations per device
- Error state responsive handling

OUTPUT: Comprehensive responsive design guide showing how Waajal Ëlëk adapts seamlessly across all device types while maintaining usability and brand consistency
```
